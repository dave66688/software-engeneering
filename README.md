# 软件开发学习仓库

> 这里是软件工程专业学生的个人学习仓库，用于记录后端开发、算法刷题与考研备考的全过程。

## 👋 个人介绍

### 个人日常与兴趣
我是一名对后端开发、算法研究感兴趣的软件工程专业学生，日常沉浸在代码逻辑的构建中 —— 对着 LeetCode 算法题反复推敲最优解，琢磨后端接口的设计逻辑与数据流转。从《计算机网络》的协议细节到《数据结构》的算法思想，都能让我感受到技术的乐趣。享受从 0 到 1 搭建后端服务、优化算法效率的过程，喜欢和同学讨论技术难题，拆解问题、找到解决方案。

### 专业技能与经验
目前虽仅掌握后端入门知识，但已扎实打下基础：
*   **语言与基础**：熟练使用 Java 语言编写基础代码，能运用面向对象思想设计简单类与接口。
*   **数据库**：掌握 MySQL 数据库基础操作，可完成数据表的创建、增删改查及简单联表查询。
*   **算法**：熟悉数据结构与算法的核心概念，能实现链表、二叉树等基础数据结构的操作，解决简单算法题。
*   **后端框架**：了解 Spring Boot 框架基本使用，可搭建基础后端项目骨架，实现简单接口开发。
*   **协作工具**：在课程实践中参与小型后端项目模块开发，使用 Git 完成代码提交、版本管理与团队协作。

### 自我评估
*   **已具备**：Java 基础编程、MySQL 基础操作、数据结构与算法基础、Spring Boot 入门、Git 协作基础。
*   **感兴趣方向**：后端服务开发、算法优化、分布式系统基础认知。
*   **待提升**：Spring Cloud/Dubbo 等框架深度应用、数据库优化与高并发处理、算法实战能力。

## 🎯 未来三年发展规划（考研方向）

### 核心目标
全力备战考研，往后端与算法方向深耕，毕业后成长为具备扎实技术能力的后端开发或算法工程师。

### 备考理由
1.  对后端技术底层逻辑、算法优化有强烈探索欲，目前基础仅能满足入门需求，希望通过考研系统学习深层技术知识。
2.  考研能接触更优质的学术与实践资源，在导师指导下参与科研项目，实现从“入门者”到“专业人才”的跨越。

### 具体规划
*   **大三**：启动考研备考，系统梳理专业课；跟随老师完成 1-2 个完整后端项目，参与算法竞赛，积累实战经验。
*   **大四**：全力冲刺考研，强化专业课、数学、英语复习与真题训练；整理项目与竞赛经历，为复试做准备。
